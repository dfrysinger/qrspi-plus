   `message`: `Task 2's splitter trailing-newline normalization (lines 386-388) is broken by command substitution semantics: `[[ $(tail -c1 "$out") != $'\n' ]]` always evaluates true because `$(...)` strips the trailing newline before the comparison runs, so `tail -c1` of a file ending in `\n` is captured as the empty string, which is `!= $'\n'`, so the script appends a SECOND `\n`. Resulting per-finding files end in `\n\n`, not "exactly one `\n`" as the per-finding contract specifies (spec §1, `## skills/reviewer-protocol/SKILL.md`: "Trailing newline — every per-finding file ends with exactly one `\n`"). Fix: replace the `$(tail -c1 …)` form with `tail -c1 "$out" | od -An -c | grep -q '\\n'` (or equivalent), or use `python3 -c` / `printf` based byte-precise check; alternatively, write the awk segment such that exactly one trailing `\n` is guaranteed and skip the post-hoc fix-up.`
